HTML

  • The <head> element contains infomation about the webpage.
  • The <body> element represents the visible content to show to the user.
  • <header>, <footer>, <main>, <aside>, <nav>, <section>, <article> are known as semantic elements. They work the same as <div> but are easier to read and identify the parts of a webpage.
  • Use the <code> tag to better identify inline code examples.
  • Use the <pre> tag to define a block of code. This code will be formatted with monospace font where the contents cannot be formatted and line breaks are recognized without using <br/> elements.
  • Use the <p> tag to define a paragraph of text.
  • Use the <h1> through <h6> tags to define up to six levels of headers from large to small.
  • There are three kinds of lists: unordered, ordered, and defintion.
    • unordered lists, or bulletted lists, are defined with the <ul> tag.
    • ordered lists, or numbered lists, are defined with the <ol> tag.
    • defintion lists are defined with the <dl tag. Unlike the <ul> and <ol> tags that use the <li> tag for list items, the <dl> tag uses <dt> for definion terms and <dd> and definition descriptions.
  • Lists can be nested, as seen above in that previous point.
  • List items (<li>) can be consecutively used.
  • Defintion lists are underappreciated. You can use <dd> consecutively to describe the most recently <dt> that was used. You should use <dl> elements for terms and defintions over the other types of lists, and unless you are explicitly writing about something should use this instead of a header and a bunch of paragraphs.
  • The <dfn> element is a tag that denotes a defintion. It might look italic, but it does not have the same meaning as em which is used for italics.
  • <strong> and <em> are recommended over <b> and <i> for bold and italic text, respectively.
  • &lt;, &gt;, and &amp; are escape entities used in HTML for less than (<), greater than (>), and the ampersand (&). You should use these entities when writing about HTML tags. There's also &quot; and &apos; for double (") and single (') quotes
  • Most HTML tags fall into two cagetories: block elements and inline elements.
  • The most generic block element is the <div> (division); the most generic inline element is the <span> (span); Both elements are described as non-semantic elements.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • If you are going to use box-shadow, include a blur-radius to make your shadows look better.

Git

  • git status checks what branch we are currently on.
  • git checkout -b branch-name creates a new branch and switches to it.
  • git add -A commits all files to the working branch.
  • git commit -m "added code to HTML file" Make a commit with a message.
  • git pull origin main make sure we have pulled the latest data from the main branch.
  • git push origin feature/add-html push all of our changes to the feature/add-html branch.
